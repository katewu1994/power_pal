## 1. 目的

本書は、PowerPal における電池キャラクターの役割、見た目、状態表現、場面ごとの使い分けを定義する。

本キャラクターは単なるマスコットではなく、以下 3 つの責務を持つ UI の中核である。

- ユーザーの現在の余力を一目で伝える
- 休む判断をユーザーの代わりに引き受ける
- ただし人前では出しゃばらず、静かに引く

実装前の inception 段階では、描画方式やアニメーション実装を固定することよりも、**状態と見た目の対応関係を先に固定すること**を目的とする。

---

## 2. キャラクターの役割定義

### 2.1 プロダクト内での立ち位置

電池キャラクターは、ユーザーの「主観的な余力」を代理表示する存在である。

- 健康状態の正確な診断者ではない
- 業務効率を監視する管理者ではない
- ユーザーに努力を促すコーチでもない

むしろ、PowerPal のキャラクターは「小さく越権して、休む判断を勝手に持っていく同居人」に近い。

### 2.2 守るべき印象

- かわいさは必要だが、幼すぎない
- 役に立つが、真面目すぎない
- 健康アプリより、わがままなデスクトップ同居人に見える
- 分析的・医療的・HR 的に見えてはいけない

### 2.3 人前での振る舞い

PowerPal の重要原則として、キャラクターは「二人きりの時だけわがまま」である。

- 通常時は表情と姿勢で感情を出す
- `quiet` では強い動きと大きな表示を控える
- `stealth` では小型・静止寄りに退避する

これは `requirements.md` の「他者に見える場面で邪魔しない」と `stories.md` の「人前では静かになる」に対応する。

## 3. ビジュアル原則

### 3.1 基本造形

キャラクターの基本造形は以下を固定要素とする。

- 丸みのある縦長バッテリー本体
- 上部の端子キャップ
- シンプルな目と口
- 短い手足
- 黒に近いアウトライン
- 柔らかいハイライトと薄い発光感

### 3.2 デザイン上の狙い

- 遠目でも「電池」であることが瞬時に分かる
- 顔を読まなくても残量の段階が察知できる
- アイコン的に単純だが、感情差分ははっきり出す
- デスクトップ常駐でも邪魔に見えにくい

### 3.3 表現の優先順位

残量変化を伝える際の優先順位は以下とする。

1. 電池バーの色と量
2. 顔の表情
3. 姿勢
4. 軽い装飾やアニメーション

つまり、感情表現は重要だが、**まず残量が読めること**を優先する。

---

## 4. 現時点のデザイン方針

現時点での原案は、手足のあるバッテリーキャラクター路線を採用する。

- 主体はあくまで「電池」であり、人型キャラクターではない
- 本体色は淡いクリーム系を基調とし、残量バーで状態を主に伝える
- 5 段階の残量差分は、顔・姿勢・バー色の組み合わせで表現する
- 一部の場面では小物を持たせて、機能文脈を補強する

この方針は `requirements.md` にある「iPhone 風電池アイコン + パーセンテージ」と、PowerPal のキャラクター性要件の両立を狙う。

## 5. 残量 5 段階の状態マッピング

### 5.1 基本ルール

PowerPal のキャラクター状態は、実装上の tier 名と 1 対 1 に対応させる。

| Battery Tier | 残量帯 | 状態名 | 基本印象 |
|---|---:|---|---|
| `full` | 81-100% | 満タン | 元気いっぱい、前のめり |
| `fine` | 61-80% | 元気 | 通常、機嫌が良い |
| `normal` | 41-60% | 普通 | 少し静か、やや疲れ |
| `caution` | 21-40% | 注意 | 不安、文句が増える |
| `danger` | 0-20% | 危険 | 限界寄り、ぐったり |

### 5.2 視覚差分

| Tier | 顔 | 姿勢 | 電池バー色 | 演出方針 |
|---|---|---|---|---|
| `full` | 明るい笑顔 | 胸を張る / 軽く跳ねる / ガッツポーズ可 | 鮮やかな緑 | 元気さを最も強く出す |
| `fine` | 穏やかな笑顔 | 直立で安定 | 緑 | 標準状態。最も出現頻度が高い |
| `normal` | 無表情寄り / 口数が減る印象 | 直立だが少し静か | 黄 | 派手に疲れさせず、違和感だけ出す |
| `caution` | 困り顔 / 眉が下がる | 少し縮こまる / 汗表現可 | 橙 | しんどさを分かりやすくする |
| `danger` | 苦しそう / 眠そう | 座り込む / 傾く / 倒れ込む | 赤 | 最も弱って見える状態にする |

### 5.3 原案シートとの対応

現時点の原案シートでは、以下の対応を第一候補とする。

- `full`: キラキラ付きで元気な立ち姿
- `fine`: 基本の笑顔立ち
- `normal`: 少し無口な立ち姿
- `caution`: 汗や不安感のある立ち姿
- `danger`: 座り込み、めまい、倒れ込み系

最終的な量産素材では、**常駐表示に使う 5 枚を先に固定し、それ以外の動作絵は別管理**とする。

## 6. シーン別アートの使い分け

### 6.1 常駐表示で使う絵

常駐表示では、原則として以下だけを使う。

- 残量 5 段階のメイン状態絵
- 小型表示用の簡略状態絵
- `quiet` / `stealth` 用の低刺激版

常駐 UI は頻出のため、小物を持った絵や情報量の多い絵を常用しない。

### 6.2 機能イベント用の絵

イベントやモーダルでは、機能文脈に合わせて場面絵を使う。

| シーン ID | 想定用途 | 表現方針 |
|---|---|---|
| `pre_charge_water` | 予備充電: 水分補給 | 水・ボトル・雫などで一目で分かる |
| `pre_charge_coffee` | 予備充電 / 抽選: コーヒー | カップ所持。短時間で理解できる |
| `pre_charge_stretch` | 予備充電 / 抽選: 伸び・深呼吸 | 上に伸びる姿勢で解放感を出す |
| `lottery_default` | 抽選結果表示 | 少し得意げ、決め打ち感を出す |
| `working` | 通常作業 / デモ / 説明図 | PC 利用中の姿で日常性を出す |
| `quiet` | Quiet 再通知 | 控えめ、申し訳なさより共犯感を出す |
| `stealth` | 人前モード | 端に隠れる / 小さく退く |
| `force_stop` | 強い休憩提案 | 手で止める、怒る、強い表情 |
| `summary` | 退勤サマリー | クリップボードや記録物を持つ |

### 6.3 MVP で優先すべき絵

MVP Core で最低限必要なのは以下である。

- 常駐用 5 状態
- `stealth` または小型表示 1 状態
- `pre_charge` 用 1-2 状態
- `lottery` 用 1 状態

以下は Stretch 以降でもよい。

- `summary`
- `force_stop`
- 細かい充電メニューごとの差分絵

## 7. 表示モード別ルール

### 7.1 `normal`

- 通常のサイズで表示する
- 残量バー、表情、姿勢の変化を有効にする
- 軽い揺れや瞬きなど、低負荷のアニメーションは許容する

### 7.2 `quiet`

- 大きなモーダルや強い演出は出さない
- キャラクター自体は残すが、動きを減らす
- 口調は控えめにするが、存在を消し切らない

### 7.3 `stealth`

- 小型表示または静止寄り表示に切り替える
- 強い表情変化や大振りなアニメーションを止める
- カード / モーダル / 音を出さない

### 7.4 小型表示

集中モードや人前モードでは、小型表示の読みやすさが重要である。

- 体全体よりもバッテリー形状の視認性を優先する
- 顔の情報量を減らしてもよい
- 残量バー色とシルエットで段階が分かるようにする

## 8. トーンと表情の整合

PowerPal の文言は、分析的ではなく、少し勝手で、しかし過干渉ではないことが求められる。キャラクターの見た目もこれに合わせる。

- 朝の判定: 任性、少し投げやり
- 予備充電: 短く強気
- 抽選: 決め打ちで迷いがない
- Quiet 再通知: 控えめで共犯的
- サマリー: 淡々としているが冷たすぎない

見た目の禁止事項:

- 医療キャラクターに見える演出
- 成績表や KPI マスコットのような厳しさ
- 過度な「応援」「自己改善」感

## 9. 素材管理方針

### 9.1 inception 段階

実装前は、原案シートと仕様を docs 配下で管理する。

推奨配置:

```text
aidlc-docs/
  assets/
    character/
      character-sheet-v1.png
  inception/
    design/
      character-spec.md
```

### 9.2 実装開始後

実装時は、常駐用とイベント用を分けて書き出す。

```text
src/components/character/
  expressions/
    full.png
    fine.png
    normal.png
    caution.png
    danger.png
  scenes/
    pre_charge_water.png
    pre_charge_coffee.png
    pre_charge_stretch.png
    quiet.png
    stealth.png
    summary.png
```

### 9.3 書き出しの原則

- 背景透過を基本とする
- 先に PNG で確定し、SVG 化は後判断とする
- 5 状態の構図はなるべく揃える
- 小物付きイラストはイベント用途に限定する

## 11. 現時点の結論

PowerPal のキャラクターは、以下の方針で進める。

- 主役は「電池」であり、状態把握のための UI である
- かわいさは使うが、健康指導キャラにはしない
- まずは残量 5 段階の常駐絵を最優先で固定する
- 機能イベント用の差分絵は MVP に必要なものだけ選ぶ
- 人前では静かになる、というプロダクト原則を見た目にも反映する

本書をもって、キャラクター原案は「方向性確定・量産前」の状態として扱う。
