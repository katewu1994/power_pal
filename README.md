# PowerPal

AWS Summit Japan 2026 AI-DLC ハッカソン 参加作品

- Team: サボりマスター
- Product: PowerPal
- Status: Hackathon MVP の企画・設計ドキュメントを収録中

> 休む判断まで、手放そう。退勤時に、まだ少し余力を残すために。

忙しいデスクワーカーは、自分の消耗に気づいたときには、休む判断そのものが負担になっています。

PowerPal は、会議密度や作業状況からデスクワーカーの消耗を推定し、休憩の判断と充電行動の提案を電池キャラクターが肩代わりするデスクトップ常駐アシスタントです。

一般的な休憩リマインダーのように自己管理を求めるのではなく、「休むかどうか」「何をして休むか」「今どれくらい消耗しているか」を、キャラクターに少し越権的に委ねる体験をつくります。

## ペルソナの痛み

PowerPal が向き合うのは、意志が弱い人ではなく、仕事中に自分の身体を後回しにしてしまうデスクワーカーです。

たとえば、午後に 3〜5 件の会議が詰まるエンジニアや、13:00-17:00 の連続会議を水も飲まずに走り切ってしまう PM。本人は怠けたいわけではありません。ただ、集中と会議のあいだで、次のような小さな判断が積み重なっていきます。

- いま疲れているのか、まだ大丈夫なのか。
- 会議前にトイレへ行くべきか。
- 水を飲むべきか、コーヒーを入れるべきか。
- 10 分の空き時間をどう使えば回復できるのか。
- いま休むのは甘えなのか、それとも必要な補給なのか。

この「小さな自己判断」こそが、忙しい日の最後に効いてきます。PowerPal は、その判断疲れを電池君に預けるためのプロダクトです。

## 企画目的

ハッカソンテーマ「人をダメにする」に対して、PowerPal は「仕事をサボらせる」「人間関係を代行する」という方向ではなく、もっと静かでクリーンなダメさを狙います。

それは、ユーザーが自分で身体状態や休憩タイミングを判断しなくなることです。

PowerPal は、外部の世界を勝手に動かしません。会議を断らない。同僚に返信しない。予定を変更しない。画面共有や会議中に飛び出さない。代わりに、ユーザー本人の目、水分、トイレ、カフェイン、注意力、元気だけを、少し過保護に管理します。

つまり、PowerPal が人をダメにする方法は、堕落ではなく「甘やかし」です。

## 1 分で分かる

- 連続会議の前に、消耗を予測して先回りで「予備充電」を提案します。
- 休憩内容はユーザーに考えさせず、電池君がその場で充電メニューを抽選します。
- 会議中や画面共有中は静かに隠れ、低電量時だけ少し強めに介入します。

## アイデアピッチ

**PowerPal は、休む判断まで肩代わりするデスクトップ常駐のしごと電池です。**

朝、ユーザーが自分で今日の調子を選ぶ必要はありません。PowerPal は、昨日の働き方、今日の会議密度、天気、そして少しだけわがままな電池君の直感から、今日の初期電量を判定します。

> 今日は雨。3 目盛り。理由はそれだけ。

午後に長い連続会議がある日は、PowerPal は先に「予備充電モード」に入ります。

> 13:00 から会議地獄。12:40 までに水・トイレ・コーヒー。これは提案ではなく、生存戦略。

ユーザーが「充電する」を押すと、電池君がその場で充電メニューを抽選します。自分で選ぶ必要はありません。

> 今日の充電: 外の空気 5 分。行ってきて。

不満なら再抽選できます。ただし、毎日 3 回まで。ユーザーには少しだけ余白がありますが、主導権はもう電池君の側にあります。

低電量のまま働き続けると、電池君は少し強めに止めます。

> ダメ。今日のあなたには判断する権利ない。まず水。

ただし、電池君には絶対の境界があります。

**人前では邪魔しない。二人きりの時だけ、わがまま。**

PowerPal は健康管理アプリではありません。働く人が「休むべきかどうか」すら自分で判断しなくてよくなる、甘やかし型のデスクトップ相棒です。

目標はひとつだけです。

> 退勤後も、まだ少し元気が残っているように。

## デモ

1. 朝、電池君が「今日は3目盛りスタート」とひと言告げる
   当日の予定や前日の状態から、電池君が今日の初期電量を決める。
2. 会議が詰まる前に、電池君が先手で動く
   連続会議の 15 分前、ユーザーが気づく前に予備充電を促す。
3. 電池君が充電メニューを決める。ユーザーは選ばない
   「充電する」を押したら、電池君がその場でやることを宣告する。
4. 本当に消耗したとき、電池君は少し強くなる。人前では静かに消える
   低電量では介入を強め、会議中・画面共有中は自動で隠れる。

## 設計の境界

- ユーザー本人の身体と注意力には越権的に関わる。
- 会議・同僚・上司・顧客など、職場の人間関係には介入しない。
- カレンダー連携は read-only 前提で、個人データはローカルファーストで扱う。
